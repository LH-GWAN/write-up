# SSRF
1. 코드를 확인해보면 `socket` 라우트에서 사용자가 입력한 `host`, `port` 값을 이용해 TCP 소켓 연결을 생성한다.
2. `SOCK_STREAM`은 TCP 연결을 의미하고, `sendall(data.encode())`를 통해 사용자가 입력한 `data`가 그대로 대상 서버로 전송된다.
3. 즉, 단순한 문자열뿐만 아니라 HTTP 요청 형식의 데이터도 직접 작성하여 보낼 수 있다.
4. 또한 `admin` 라우트는 헤더나 쿠키 userid 같은 값들을 확인한다.
5. 직접 `admin`에 접근은 불가능하지만, `socket` 기능을 이용해 서버가 자기 자신(`127.0.0.1:8000`)으로 요청을 보내게 만들면 조건을 만족할 수 있다.
6. 따라서 `socket`의 `host`에는 `127.0.0.1`, `port`에는 `8000`을 입력하고, `data`에는 `admin`으로 향하는 HTTP POST 요청을 직접 작성한다.
7. 이때 코드에서 검사하는 헤더와 쿠키, form 데이터를 모두 포함해야 한다.
8. 결과적으로 사용자가 직접 브라우저에서 `admin`에 접근하는 것이 아니라, 서버 내부에서 `/admin`으로 요청을 보내게 만들어 admin 조건을 만족시키는 방식으로 FLAG를 획득할 수 있었다.
