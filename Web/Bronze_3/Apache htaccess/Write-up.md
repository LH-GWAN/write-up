# .htaccess를 이용한 파일 업로드 우회
1. 파일 업로드 기능이 존재하고, 업로드된 파일에 직접 접근할 수 있으므로 파일 업로드 취약점 가능성을 확인했다.
2. 소스코드를 분석한 결과 php, php3, php4, php5, pht, phtml 등의 확장자는 차단되고 있었다.
3. 대문자 .PHP나 .jpg, .txt 등의 확장자는 업로드할 수 있었지만, 서버가 해당 파일을 PHP로 해석하지 않아 웹셸로 실행되지는 않았다.
4. 업로드 차단 목록에 Apache 설정 파일인 .htaccess가 포함되어 있지 않은 점을 확인했다.
5. .htaccess 파일에 다음 설정을 작성하여 .txt 확장자를 PHP 파일로 처리하도록 설정했다.
6. 이후 PHP 웹셸 코드가 들어 있는 .txt 파일을 업로드했다.
7. 업로드된 웹셸에 ?cmd=ls /를 전달하여 루트 디렉터리의 파일 목록을 확인했다.
8. /flag 실행 파일을 확인한 뒤 ?cmd=/flag를 전달하여 플래그를 획득했다.
